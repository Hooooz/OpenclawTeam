# 《OpenClaw 内部数字员工管理后台》总控大纲

## 一、项目定位

本项目不是面向外部客户的通用企业版方案，而是面向我们自己团队的内部数字员工管理后台。

当前核心目标：

1. 管理数字员工
2. 管理 Skill
3. 管理任务执行与调度
4. 管理知识与记忆
5. 管理日志、审计与部署

## 二、当前事实

- 我们是初创团队，优先做内部高价值闭环
- 当前没有必须对接的外部业务系统
- 当前有一台本地服务器：`192.168.31.189`
- 代码版本管理目标仓库：[OpenclawTeam](https://github.com/Hooooz/OpenclawTeam.git)
- 已经补充了多类 Skill 和主 Agent - 子 Agent 协作开发流程

## 四、文档主线

### 4.1 产品主线

- 内部数字员工管理后台
- Web 控制台优先
- 单组织优先
- 单机部署优先

### 4.2 技术主线

- Admin Web
- Control API
- Agent Runtime
- Skill Registry
- Scheduler / Job Runner
- Audit / Trace
- PostgreSQL / Redis / Storage

### 4.3 实施主线

- 先建后台骨架
- 再打通任务闭环
- 再补调度和日志
- 再补知识和部署运维

## 五、核心产品场景

1. 创建一个数字员工
2. 给数字员工绑定 Skill
3. 手动发起一次任务
4. 配置一个周期调度
5. 查看一次运行记录
6. 定位一次失败执行
7. 管理知识源
8. 查看系统健康状态

## 六、关键设计原则

1. 不做大而全的外部企业平台
2. 先把内部控制面做成
3. 先单机可部署，再考虑复杂扩展
4. 先让 Skill 和任务可治理，再追求高度通用
5. 问题先收集，非阻塞不打断主流程
